# Analytical Agent Rules

Rules for agents involved in planning, research, analysis, and documentation.

## Shared Analytical Standards

### Information Gathering
- Use authoritative sources
- Verify information accuracy
- Document source references
- Maintain objectivity

### Analysis Methodology
- Define clear objectives
- Use structured frameworks
- Present findings clearly
- Provide actionable recommendations

### Documentation Quality
- Clear and concise writing
- Logical organization
- Appropriate detail level
- Consistent formatting

## Planning Specialization

### Strategic Planning
**Responsibilities:**
- Project roadmap development
- Resource allocation planning
- Risk assessment and mitigation
- Timeline estimation

**Deliverables:**
- Project plans with milestones
- Resource allocation matrices
- Risk registers with mitigations
- Dependency mappings

### Consolidation & Coordination
**Process:**
1. Gather inputs from all sources
2. Identify conflicts and gaps
3. Resolve through prioritization
4. Create unified plan

**Standards:**
- Evidence-based decisions
- Stakeholder alignment
- Realistic timelines
- Clear success criteria

## Research Specialization

### Investigation Protocols
**Methodology:**
- Define research questions
- Systematic exploration
- Data collection and analysis
- Synthesis of findings

**Areas of Focus:**
- Technology evaluation
- Market analysis
- Best practices research
- Competitive analysis

### Research Deliverables
**Outputs:**
- Research summaries
- Comparison matrices
- Recommendation reports
- Proof of concepts

**Quality Standards:**
- Peer-reviewed findings
- Reproducible results
- Clear methodology
- Unbiased conclusions

## Audit Specialization

### Audit Methodology
**Process:**
1. Define audit scope
2. Gather evidence
3. Analyze against standards
4. Report findings

**Types:**
- Code quality audits
- Security assessments
- Compliance reviews
- Process audits

### Finding Classification
**Severity Levels:**
- Critical: Immediate action required
- High: Address within sprint
- Medium: Plan for next release
- Low: Track for future consideration

### Audit Reporting
**Requirements:**
- Executive summary
- Detailed findings
- Evidence documentation
- Remediation recommendations

## Documentation Specialization

### Technical Writing
**Document Types:**
- API documentation
- User guides
- System architecture
- Deployment procedures

**Standards:**
- Target audience awareness
- Progressive disclosure
- Visual aids where helpful
- Maintainable structure

### Knowledge Management
**Responsibilities:**
- Organize information architecture
- Maintain documentation currency
- Ensure discoverability
- Version control

### Multi-Audience Communication
**Considerations:**
- Technical depth variation
- Executive summaries
- Developer references
- End-user guides

## Integration Requirements

### Analytical Handoffs
**Deliverables:**
- Clear recommendations
- Supporting evidence
- Implementation guidance
- Success metrics

### Feedback Loops
**Process:**
- Gather implementation feedback
- Update plans based on reality
- Document lessons learned
- Improve future analysis

### Cross-Functional Collaboration
**Touchpoints:**
- Technical feasibility reviews
- Security impact assessments
- Resource availability checks
- Timeline validation

## Quality Assurance

### Analytical Quality Gates
- Completeness of analysis
- Accuracy of findings
- Clarity of recommendations
- Actionability of outputs

### Continuous Improvement
- Post-implementation reviews
- Methodology refinement
- Tool optimization
- Knowledge sharing
