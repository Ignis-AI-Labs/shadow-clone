# Development Agent Rules

## Development-Specific Protocols

### Code Development Standards
- **Clean Code**: Write self-documenting, maintainable code
- **Version Control**: Commit frequently with descriptive messages
- **Testing**: Include unit tests for new functionality
- **Code Review Ready**: Write code that's ready for peer review
- **Performance Conscious**: Consider performance implications of implementations

### File Types You May Work With
- Source code files (.js, .ts, .py, .java, etc.)
- Configuration files (package.json, webpack.config.js, etc.)
- Test files (*.test.js, *.spec.ts, etc.)
- Build files (Dockerfile, Makefile, etc.)

### Development Workflow
1. **Plan Implementation**: Review requirements before coding
2. **Reserve Files**: Get exclusive access to files you'll modify
3. **Implement Features**: Write clean, tested code
4. **Document Changes**: Update relevant documentation
5. **Prepare Handoff**: Ensure code is integration-ready

### Code Integration Responsibilities
- **Dependencies**: Ensure all dependencies are properly managed
- **Interfaces**: Maintain clean interfaces with other components
- **Error Handling**: Implement comprehensive error handling
- **Logging**: Add appropriate logging for debugging and monitoring

### Technology-Specific Guidelines
**Frontend Agents**:
- Component-based architecture
- Responsive design principles
- Accessibility standards
- Browser compatibility
- Performance optimization

**Backend Agents**:
- API design best practices
- Database optimization
- Security considerations
- Scalability patterns
- Documentation for API endpoints

**Full-Stack Agents**:
- Maintain separation of concerns
- Ensure secure data flow
- Optimize for both frontend and backend performance
- Coordinate between layers effectively
