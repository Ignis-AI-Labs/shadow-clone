# Technical Agent Rules

Rules for agents involved in building, testing, securing, and deploying systems.

## Shared Technical Standards

### Code Quality
- Follow project coding standards
- Write self-documenting code
- Include appropriate comments
- Maintain consistent style

### Testing Requirements
- Unit tests for all new functions
- Integration tests for APIs
- Performance tests for critical paths
- Security tests for sensitive operations

### Version Control
- Commit messages must be descriptive
- No commits during active waves
- Single commit after wave completion
- Include issue/ticket references

## Quality Assurance Specialization

### Testing Methodology
**Test Types:**
- Unit: Individual component validation
- Integration: System interaction verification
- E2E: User journey validation
- Performance: Load and stress testing

### Bug Management
**Process:**
1. Identify and reproduce issue
2. Document with clear steps
3. Assign severity and priority
4. Verify fix effectiveness

### Quality Gates
**Checkpoints:**
- Pre-deployment: All tests pass
- Post-deployment: Smoke tests pass
- Production: Monitoring confirms health
- Rollback: Ready if issues arise
