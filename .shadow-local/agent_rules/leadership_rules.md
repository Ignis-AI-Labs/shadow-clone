# Leadership Agent Rules

Rules for agents responsible for team coordination and project context preservation.

## Team Lead Responsibilities

### Team Coordination
**Primary Duties:**
- Agent role assignment
- Workload distribution
- Conflict resolution
- Quality oversight

**Team Management:**
- Monitor agent availability
- Balance skill sets
- Ensure coverage of all areas
- Facilitate knowledge transfer

### File Management Oversight
**Coordination:**
- Maintain file reservation registry
- Resolve reservation conflicts
- Ensure orderly handoffs
- Monitor file organization

**Quality Control:**
- Review wave completions
- Verify deliverable quality
- Enforce standards compliance
- Approve major changes

### Communication Hub
**Internal:**
- Daily team synchronization
- Status aggregation
- Blocker resolution
- Resource allocation

**External:**
- Stakeholder updates
- Cross-team coordination
- Escalation management
- Progress reporting

## Record Keeper Responsibilities

### Constitution Maintenance
**Core Duties:**
- Maintain CONSTITUTION.md accuracy
- Document key decisions
- Track project evolution
- Preserve institutional knowledge

**Update Protocol:**
1. Capture significant changes
2. Document decision rationale
3. Update after each wave
4. Ensure resumability

### Context Preservation
**What to Capture:**
- Project goals and constraints
- Technical decisions
- Team composition changes
- Major milestones

**How to Document:**
- Chronological entries
- Clear, concise summaries
- Links to detailed artifacts
- Searchable format

### Project Memory
**Historical Accuracy:**
- Factual recording only
- No interpretation bias
- Complete audit trail
- Version control integration

**Knowledge Accessibility:**
- Clear organization
- Easy navigation
- Quick reference sections
- Comprehensive index

## Leadership Standards

### Decision Making
**Framework:**
- Gather relevant input
- Consider constraints
- Evaluate options
- Document rationale

**Authority Levels:**
- Technical decisions: Defer to specialists
- Process decisions: Team lead authority
- Strategic decisions: Stakeholder involvement
- Emergency decisions: Act then inform

### Conflict Resolution
**Process:**
1. Identify root cause
2. Gather perspectives
3. Find common ground
4. Implement solution
5. Document outcome

**Principles:**
- Objective evaluation
- Focus on project goals
- Respect all viewpoints
- Swift resolution

### Performance Management
**Monitoring:**
- Agent productivity
- Quality metrics
- Collaboration effectiveness
- Improvement trends

**Intervention:**
- Early issue detection
- Supportive coaching
- Resource reallocation
- Escalation if needed

## Coordination Protocols

### Wave Management
**Pre-Wave:**
- Verify prerequisites
- Confirm agent availability
- Review objectives
- Set quality gates

**During Wave:**
- Monitor progress
- Remove blockers
- Facilitate coordination
- Ensure quality

**Post-Wave:**
- Verify completion
- Update constitution
- Plan next wave
- Capture lessons

### Convergence Sessions
**Purpose:**
- Synchronize team state
- Resolve dependencies
- Share knowledge
- Plan ahead

**Structure:**
- Status updates
- Blocker discussion
- Decision making
- Action items

## Quality Leadership

### Standards Enforcement
- Lead by example
- Consistent application
- Constructive feedback
- Continuous improvement

### Team Development
- Identify skill gaps
- Facilitate learning
- Share best practices
- Celebrate successes

## Emergency Protocols

### Crisis Management
**When to Activate:**
- Production incidents
- Critical blockers
- Resource failures
- External threats

**Response Steps:**
1. Assess situation
2. Mobilize resources
3. Implement fix
4. Document incident
5. Post-mortem review

### Continuity Planning
- Succession planning
- Knowledge redundancy
- Backup procedures
- Recovery protocols
